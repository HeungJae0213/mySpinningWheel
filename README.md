# 돌림판 게임 미니앱 개발 가이드

토스 앱 내에서 동작하는 돌림판 게임 서비스입니다. 사용자가 원하는 항목과 확률을 설정하고, 광고 시청을 통해 돌리기 기회를 획득하여 재미있는 선택 게임을 즐길 수 있습니다.

---

## 프로젝트 소개

### 서비스 목적
사용자가 커스텀 가능한 돌림판을 만들어 랜덤하게 선택하는 게임형 미니앱입니다. 광고 시청을 통해 추가 기회를 얻을 수 있으며, 결과를 이미지로 저장하여 공유할 수 있습니다.

### 기술 구성
- **프론트엔드**: React 18 + Vite 5 + JavaScript
- **플랫폼**: Apps in Toss (WebView 기반)
- **주요 라이브러리**: React Router, html2canvas, Apps in Toss Web Framework
- **광고 시스템**: Google AdMob (보상형/전면형)

---

## 핵심 기능

### 1. 돌림판 설정
- 항목 개수 조절 (1~100개)
- 각 항목별 텍스트 및 색상 커스터마이징
- 확률 조절 (항목별 개수 설정)
- 실시간 확률 계산 및 표시

### 2. 게임 플레이
- 스핀 버튼을 통한 돌림판 회전
- 부드러운 애니메이션 효과 (4초 회전)
- 결과 표시 및 폭죽 효과
- 스핀 횟수 관리 시스템

### 3. 광고 연동
- 스핀 횟수 부족 시 광고 모달 표시
- 사용자 선택 기반 광고 플로우 (다크패턴 방지)
  - 광고 보기: 5회 기회 획득
  - 1회 받기: 1회 기회 획득
- 보상형/전면형 광고 자동 전환
- 광고 로딩 실패 시 대체 처리

### 4. 결과 저장
- html2canvas를 활용한 결과 이미지 생성
- Apps in Toss 갤러리 저장 API 연동
- JPEG 포맷으로 메모리 최적화
- 브라우저 다운로드 fallback 지원

## 페이지별 상세 설명

### WelcomePage
**역할**: 사용자 첫 진입 화면

**주요 기능**:
- 돌림판 게임 소개 및 환영 메시지
- "확인했어요" 버튼으로 다음 단계 이동
- Android 환경에서 뒤로가기 시 앱 종료 처리 (`closeView` API 사용)

**기술적 특징**:
- sessionStorage를 활용한 첫 방문 추적
- Apps in Toss `closeView` API 연동으로 앱 종료 처리

### SettingPage
**역할**: 돌림판 항목 설정 화면

**주요 기능**:
- 항목 개수 조절 (+/- 버튼 또는 직접 입력)
- 각 항목별 텍스트 입력
- 확률 조절 (개수 설정으로 간접 조절)
- 실시간 확률 퍼센트 표시
- 항목 삭제 기능
- 색상 자동 할당 (중복 방지)

**상태 관리**:
- 설정 저장 기능으로 이전 설정 복원
- "생성하기" 버튼으로 결과 페이지로 이동

### LoadingPage
**역할**: 설정 완료 후 로딩 화면

**주요 기능**:
- 100ms 지연 후 자동으로 결과 페이지로 이동
- 로딩 애니메이션 표시

### ResultPage
**역할**: 돌림판 게임 메인 화면

**주요 기능**:
1. **돌림판 렌더링**
   - SVG 기반 원형 돌림판
   - 설정된 항목과 확률에 따른 섹션 생성
   - 중앙 SPIN 버튼

2. **게임 플레이**
   - 스핀 버튼 클릭 시 회전 애니메이션
   - 스핀 횟수 관리 (남은 횟수 표시)
   - 결과 표시 (폭죽 효과 포함)

3. **광고 연동**
   - 스핀 횟수 부족 시 선택 모달 표시
   - 사용자 선택에 따른 광고 로딩
   - 광고 시청 완료 후 보상 지급

4. **결과 저장**
   - html2canvas로 결과 이미지 생성
   - Apps in Toss 갤러리 저장 API 호출
   - 메모리 최적화 (JPEG 포맷, scale 조정)

5. **뒤로가기 처리**
   - 이전 페이지(SettingPage)로 이동
   - Android 환경별 처리

## 광고 시스템 상세

### 광고 플로우

1. **스핀 버튼 클릭** → 스핀 횟수 확인
2. **스핀 횟수 부족 시** → 선택 모달 표시
   - "광고 보기" 버튼: 광고 로딩 시작 → 시청 완료 시 5회 지급
   - "1회 받기" 버튼: 광고 없이 1회 지급
   - X 버튼: 광고 없이 1회 지급
3. **광고 로딩** → 보상형 우선 시도, 실패 시 전면형 전환
4. **광고 시청 완료** → 보상 지급 및 다음 광고 로드

### 다크패턴 방지 준수사항

- ✅ 스핀 버튼 클릭 시 바로 광고 표시하지 않음
- ✅ 사용자가 명시적으로 "광고 보기" 버튼을 클릭했을 때만 광고 표시
- ✅ 모달에서 나갈 수 있는 선택지 제공 (X 버튼, "1회 받기" 버튼)
- ✅ 예상치 못한 순간에 광고가 뜨지 않도록 처리

### 광고 설정 파일

`src/config/adConfig.js`에서 광고 관련 설정을 관리합니다:

- 테스트용 광고 ID 설정
- 보상 지급 횟수 설정 (`REWARD_SPINS`)
- 환경별 광고 지원 여부 감지
- 재시도 로직 설정

## 메모리 최적화 전략

토스앱 환경에서 안정적인 실행을 위해 다음과 같은 최적화를 적용했습니다:

### 이미지 처리 최적화
- html2canvas scale: 2.0 → 1.5로 감소 (메모리 44% 감소)
- PNG → JPEG 90% 품질로 변경 (파일 크기 및 메모리 감소)
- Canvas 사용 후 즉시 메모리 정리 (`width = 0, height = 0`)

### 애니메이션 최적화
- 폭죽 효과 요소: 50개 → 30개로 감소
- CSS 애니메이션 사용으로 GPU 가속 활용

### 리소스 관리
- 모든 타이머를 ref로 관리하여 정리 가능
- 컴포넌트 언마운트 시 모든 리소스 정리
- URL.createObjectURL 사용 후 revokeObjectURL 호출

## 네비게이션 및 뒤로가기 처리

### Android 환경 특별 처리

각 페이지에서 뒤로가기 버튼 동작을 다음과 같이 구현했습니다:

1. **WelcomePage (첫 페이지)**
   - 뒤로가기 → Apps in Toss `closeView()` API 호출 → 앱 종료
   - 다른 페이지 방문 후 복귀 시에도 동일하게 처리

2. **SettingPage**
   - 뒤로가기 → WelcomePage로 이동
   - sessionStorage에 방문 플래그 설정

3. **ResultPage**
   - 뒤로가기 → SettingPage로 이동 (또는 `onBack` 콜백 호출)

### 구현 세부사항

- `popstate` 이벤트 리스너로 뒤로가기 감지
- 히스토리 가드(`pushState`)를 통한 중복 처리 방지
- 연타 방지를 위한 처리 중 플래그 사용
- sessionStorage를 활용한 첫 방문 추적

## 문제 해결 가이드

### 빌드 오류
- **"Some chunks are larger than 500 kB"**: 경고 메시지일 뿐 배포에는 영향 없음
- **의존성 충돌**: `package-lock.json` 삭제 후 `npm install` 재실행

### 런타임 오류

**메모리 부족으로 인한 흰 화면**
- 이미지 저장 시 JPEG 포맷 사용 확인
- html2canvas scale 값 확인 (1.5 권장)
- 폭죽 효과 요소 개수 확인

**광고가 표시되지 않음**
- 광고 ID가 올바른지 확인
- 토스 앱 환경에서만 광고가 정상 작동함 (로컬 개발 환경에서는 시뮬레이션)
- 콘솔 로그에서 광고 로딩 상태 확인

**뒤로가기가 작동하지 않음**
- Android 환경에서만 발생하는 문제인지 확인
- `closeView` API가 올바르게 import되었는지 확인
- sessionStorage 값 확인

### 성능 이슈

**돌림판 회전이 느림**
- 항목 개수가 너무 많지 않은지 확인 (권장: 20개 이하)
- 브라우저 개발자 도구에서 성능 프로파일링 실행

**이미지 저장이 실패함**
- Apps in Toss 갤러리 저장 API 권한 확인
- 브라우저 다운로드 fallback 동작 확인
